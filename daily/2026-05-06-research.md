### Official: Claude Code Changelog — v2.1.129 & v2.1.131 (May 6, 2026)

**Source**: https://github.com/anthropics/claude-code/releases

**Type**: Official Anthropic release notes

**What changed** (key highlights):

**v2.1.129 (May 6, 2026)**:
- **Plugin URL Installation** — `--plugin-url <url>` flag fetches plugin `.zip` from remote URLs (not just local files)
- **Synchronized Output** — `CLAUDE_CODE_FORCE_SYNC_OUTPUT=1` env var forces synchronous output on specific terminals (fixes race conditions)
- **Package Manager Auto-Update** — `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE` auto-updates via Homebrew/WinGet (zero-friction upgrade path)
- **Session Resume Improvements** — Fixed resume state corruption when restarting sessions mid-conversation
- **Terminal Behavior Fixes** — Corrected scroll-lock, output buffering, multi-line command handling

**v2.1.131 (May 6, 2026)**:
- **VS Code Extension Fix** — Fixed hardcoded build path preventing extension activation on Windows
- **Mantle Endpoint Auth** — Added missing `x-api-key` header for Mantle endpoint authentication
- **Stability** — Consolidated fixes from concurrent releases

**Real example from changelog**:

```bash
# Plugin installation from remote URL (v2.1.129)
claude plugin install --plugin-url https://my-plugin-registry.com/my-plugin.zip

# Auto-update configuration (v2.1.129)
export CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE=1
# Homebrew/WinGet now auto-updates Claude Code on new releases

# Sync output fix (v2.1.129)
export CLAUDE_CODE_FORCE_SYNC_OUTPUT=1
# Prevents output race conditions on slow/flaky terminals
```

**Problem it solves**: Plugin distribution required local `git clone` (friction) → remote ZIP URLs enable plugin marketplaces. Auto-update skips manual homebrew updates. Sync output fixes race conditions on slow terminals (WSL2, SSH, containers).
