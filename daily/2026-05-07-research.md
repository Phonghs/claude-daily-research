### Official: Claude Code Changelog — v2.1.132 (May 6, 2026)

**Source**: https://github.com/anthropics/claude-code/releases

**Type**: Official Anthropic release notes

**What changed** (key highlights):

**v2.1.132 (May 6, 22:08 UTC)** — Bedrock service tier selection, fixes for MCP, resume, and terminal handling:

- **Bedrock Integration** — Added `ANTHROPIC_BEDROCK_SERVICE_TIER` env var to select Bedrock tier (default, flex, or priority) → cost optimization for AWS deployments
- **PR Search in /resume** — Pasting PR URL into `/resume` search box now finds session that created that PR (GitHub, GitHub Enterprise, GitLab, Bitbucket) → faster session recovery
- **MCP Deduplication** — `/mcp` now shows claude.ai connectors hidden by duplicate manually-added server with same URL + hint to remove duplicate → prevents MCP conflicts
- **OpenTelemetry Numeric Attributes** — Numeric attributes on api_request/api_error log events now emitted as numbers (not strings) → better observability tooling
- **Terminal & Fullscreen Fixes** — Fixed /branch fork failures (tool_use ids without tool_result blocks), Bedrock SigV4 auth 403 errors, fullscreen mode blank after laptop sleep, pasted text starting with `/` being swallowed, mouse wheel scroll speed issues, emoji truncation in --resume

**Real example from changelog**:

```bash
# Select Bedrock service tier (reduces cost vs priority tier)
export ANTHROPIC_BEDROCK_SERVICE_TIER=flex
claude code

# Recover session by PR URL
/resume
# Paste: https://github.com/owner/repo/pull/42
# → Finds session that created PR #42 automatically

# Clean up duplicate MCP servers
/mcp
# Shows: "claude.ai connector hidden by duplicate 'https://example.com' — remove duplicate to use"
```

**Problem it solves**: Bedrock costs vary by tier (priority expensive, flex cheaper) → devs manually track settings. v2.1.132 = explicit tier env var + tooltip → optimize cost. PR search in resume = faster session recovery without manual transcript search. MCP dedup = prevents tool conflicts when connecting same service twice.
