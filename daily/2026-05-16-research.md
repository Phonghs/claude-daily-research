### 1. disler/claude-code-hooks-mastery

**URL**: https://github.com/disler/claude-code-hooks-mastery

**Type**: Hook collection / Config template

**What it actually does**:
- Input: JSON payload từ Claude Code chứa prompt text, tên tool, session ID, execution context.
- Output: Exit code (0 = thông qua, 2 = chặn) và JSON tuỳ chọn với control decision.
- Workflow: Hook kích hoạt theo thứ tự `UserPromptSubmit` → `PreToolUse` → `PostToolUse` → `Stop` — mỗi bước có script Python riêng.
- Repo demo toàn bộ 13 hook lifecycle event của Claude Code, không phụ thuộc vào quyết định của LLM để chặn hay cho phép hành động.

**Real example from README** (REQUIRED):

```python
# PreToolUse Hook — block dangerous rm commands
if re.search(r'rm\s+.*-[rf]', command):
    print("BLOCKED: Dangerous rm command", file=sys.stderr)
    sys.exit(2)  # exit code 2 blocks tool execution
```

**Install instructions**:

```bash
# Clone rồi copy vào project của bạn
cp -r .claude/hooks/ your-project/.claude/

# Cập nhật .claude/settings.json để đăng ký hook
# Mỗi hook dùng lệnh: uv run $CLAUDE_PROJECT_DIR/.claude/hooks/[name].py
```

**Problem it solves**: Tự động chặn lệnh nguy hiểm (`rm -rf`, truy cập `.env`) và ghi audit log JSON sau mỗi hook event, không cần nhấn "Allow/Deny" thủ công.

**Existing alternatives**: Manual `--allowedTools` whitelist trong Claude Code settings; `agamm/claude-code-owasp` (đã có trong dedup-set).

**Repo signals**:

- Maintainer: disler

**Solo-dev fit**: ⭐ × 5
Reason: Copy `.claude/hooks/` vào project là xong, không cần backend hay dependency phức tạp ngoài `uv`.

**Concrete action to try it**:

### 2. yoloshii/mcp-code-execution-enhanced

**URL**: https://github.com/yoloshii/mcp-code-execution-enhanced

**Type**: MCP server / SDK

**What it actually does**:
- Input: MCP server config + Python script có CLI parameters.
- Output: Kết quả thực thi tool từ MCP server, trả về dạng structured output.
- Workflow: `Discover tools` → `Viết Python script với CLI args` → `Chạy qua harness` → `Nhận kết quả` — không load toàn bộ tool schema lên context.
- Ba chế độ: Scripts với CLI args (99.6% token reduction), Direct script writing (98.7% reduction), Progressive Disclosure (lazy-load schema).

**Real example from README**:


### 4. mkXultra/ai-cli-mcp

**URL**: https://github.com/mkXultra/ai-cli-mcp

**Type**: MCP server

**What it actually does**:
- Input: Prompt text, working directory, model name (sonnet/opus/haiku/gemini/codex).
- Output: Kết quả từ AI CLI tool chạy background, trả về qua `wait` hoặc `get_result`.
- Workflow: `run` (khởi động process) → `ps` (kiểm tra status) → `result <PID>` (lấy output) — mỗi AI agent chạy độc lập với PID riêng.
- Hỗ trợ Claude, Codex, Gemini, Forge, OpenCode cùng lúc.

**Real example from README**:

```bash
# Chạy Claude task trong background
ai-cli run --cwd "$PWD" --model sonnet --prompt "summarize this repository"
ai-cli ps           # Kiểm tra processes đang chạy
ai-cli result 12345 # Lấy output theo PID
```

Hoặc qua MCP config:
```json
{
  "ai-cli-mcp": {
    "command": "npx",
    "args": ["-y", "ai-cli-mcp@latest"]
  }
}
```

**Install instructions**:

```bash
npm install -g ai-cli-mcp
# Hoặc dùng qua npx không cần install:
npx -y --package ai-cli-mcp@latest ai-cli run --cwd "$PWD" --model sonnet --prompt "hello"
```

**Problem it solves**: Cho phép Claude Code gọi các AI CLI tool khác (Gemini, Codex) như sub-agent background process — không cần mở terminal riêng hay chờ từng task xong mới chạy task tiếp theo.

**Existing alternatives**: Mở terminal riêng và chạy AI CLI thủ công; `steipete/claude-code-mcp` (đã có trong dedup-set, chỉ wrap Claude).

**Repo signals**:
- Stars: 16
- Last commit: 23/04/2026 (v2.21.0)
- Open issues: 6
- License: MIT
- Maintainer: mkXultra

**Solo-dev fit**: ⭐ × 3
Reason: `npx -y ai-cli-mcp@latest` không cần install global, nhưng cần có các AI CLI tool (claude, gemini CLI) đã cài sẵn mới hoạt động.

**Concrete action to try it**:
