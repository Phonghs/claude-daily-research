### 2. alexgreensh/token-optimizer

**URL**: https://github.com/alexgreensh/token-optimizer

**Type**: CLI tool (Claude Code plugin)

**What it actually does**:
- Input: Session data, file content, tool output, config file đang được Claude Code đọc trong session.
- Output: Token metrics per-turn, quality score từ S đến F (7-signal scoring), interactive HTML dashboard tại `localhost:24842/token-optimizer`.
- Workflow: Plugin hook intercept tool calls → detect 10 loại waste pattern (bloated config, unused skill, duplicate prompt, stuck retry loop) → áp dụng Delta Mode (chỉ gửi diff khi re-read file, ~97% savings trên changed files) + Bash Compression (16 handler nén pytest output từ 564 xuống 115 token) → Smart Compaction checkpoint session tại 20/35/50% fill → log SQLite local `~/.claude/_backups/token-optimizer/trends.db`.
- Không gửi data ra ngoài (zero network calls cho telemetry).

**Real example from README**:

```
/plugin marketplace add alexgreensh/token-optimizer
/plugin install token-optimizer@alexgreensh-token-optimizer
```

Sau đó gọi `/token-optimizer` để xem dashboard. Documented 30-day snapshot: 942 sessions, 6.13B input tokens, 82% cache hit rate, tiết kiệm "$1,500 to $2,500 per month" cho heavy Opus users.

**Install instructions**:

```bash
# Cách 1: Plugin marketplace (khuyến nghị)
/plugin marketplace add alexgreensh/token-optimizer
/plugin install token-optimizer@alexgreensh-token-optimizer

```

**Problem it solves**: README mô tả: "Your AI is getting dumber and you can't see it" — 75–85% token waste đến từ bloated config, unused skill, duplicate prompt; developer chỉ thấy được 15–25% waste qua verbose CLI output thông thường.

**Existing alternatives**: `drona23/claude-token-efficient` giải quyết verbosity ở response-level (static CLAUDE.md); `nadimtuhin/claude-token-optimizer` reorganize startup docs để giảm token lúc load (static); `alexgreensh/token-optimizer` monitor dynamic waste realtime trong session.

### 4. jztan/pdf-mcp

**URL**: https://github.com/jztan/pdf-mcp

**Type**: MCP server

**What it actually does**:
- Input: PDF file path hoặc URL truyền qua MCP tool call từ Claude.
- Output: Chunked page content, OCR-extracted text, table extraction, hybrid search results kết hợp keyword và semantic search.
- Workflow: Agent gọi `pdf_info()` để map cấu trúc tài liệu → `pdf_search()` tìm trang theo keyword → `pdf_read_pages(start, end)` fetch range cụ thể → SQLite cache persist kết quả giữa các server restart, tránh re-read.
- Semantic search cần `pip install 'pdf-mcp[semantic]'`; OCR cần Tesseract system dependency.

**Real example from README**:

```bash
pip install pdf-mcp
# Với semantic search:
pip install 'pdf-mcp[semantic]'
```

Ví dụ workflow xử lý báo cáo 200 trang: gọi `pdf_info()` → xác định trang 40–60 liên quan → `pdf_read_pages(40, 60)` → synthesize mà không load toàn bộ tài liệu vào context.

**Install instructions**:

```bash
pip install pdf-mcp
# OCR support:
# macOS: brew install tesseract
# Ubuntu: apt install tesseract-ocr
```

**Problem it solves**: Load toàn bộ PDF vào Claude context gây vượt context limit ngay cả với tài liệu trung bình (200 trang); tool này cho agent đọc chunked theo trang hoặc keyword, không cần dump toàn bộ file.

**Existing alternatives**: `@modelcontextprotocol/server-filesystem` đọc text file thẳng nhưng không có chunking hay OCR; không có MCP server nào có hybrid search + SQLite cache cho PDF theo kết quả tìm kiếm.

### 5. Hooks đọc effort level qua `$CLAUDE_EFFORT` và `effort.level` JSON field (Claude Code v2.1.141)

**Source signals**:
- Publish date: May 13, 2026
- Author: Anthropic
- Platform: Official Claude Code changelog — https://code.claude.com/docs/en/changelog (v2.1.141)

**What it actually does**:
- Input: Hook shell command trong `settings.json`; mọi hook loại PostToolUse/PreToolUse/Stop đều nhận thêm field mới.
- Output: Biến môi trường `$CLAUDE_EFFORT` chứa effort level hiện tại của session (`low`, `normal`, `high`); field `effort.level` trong JSON input của hook.
- Workflow: Hook nhận JSON input → đọc `effort.level` → branch logic tùy theo effort (chạy full test suite chỉ khi `high`, skip lint khi `low`) → không cần workaround env var thủ công như trước.

**Real example from changelog**:

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit",
      "hooks": [{
        "type": "command",
        "command": "[ \"$CLAUDE_EFFORT\" = 'high' ] && npm test || echo 'low-effort: skipping tests'"
      }]
    }]
  }
}
```

`$CLAUDE_EFFORT` cũng có thể đọc trong Bash tool commands (không chỉ hooks).

**How to apply**: Mở `.claude/settings.json` → tìm hook command đang chạy expensive operation → bọc bằng điều kiện `[ "$CLAUDE_EFFORT" = "high" ] && <expensive-cmd> || true`.

**Problem it solves**: Trước v2.1.141, hook không biết effort level của session — full test suite + linter chạy ngay cả khi user dùng `--effort low` để draft nhanh, tốn thời gian và token không cần thiết.

**Existing alternatives**: Trước v2.1.141 không có cách native nào. Workaround cũ: set custom env var thủ công (`export MY_EFFORT=low`) trước khi start Claude, nhưng không đồng bộ với `--effort` flag.

**Source signals**:
- Publish date: May 13, 2026
- Author: Anthropic engineering team
- Platform: code.claude.com/docs/en/changelog (version 2.1.141, section "New Features")

**Solo-dev fit**: ⭐ × 4  
Reason: 1 line bash condition trong existing hook, không cần install thêm gì; tiết kiệm rõ ràng khi dùng `--effort low` cho draft/brainstorm sessions.

**Concrete action to try it**:

Mở `.claude/settings.json`, tìm hook `PostToolUse` có lệnh test/lint, thêm:
```bash
[ "$CLAUDE_EFFORT" = "high" ] && npm test || echo "Skipped (low effort)"
```
