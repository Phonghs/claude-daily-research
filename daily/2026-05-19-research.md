### 2. lastmile-ai/mcp-agent

**URL**: https://github.com/lastmile-ai/mcp-agent

**Type**: SDK

**What it actually does**:
- Input: Danh sách MCP servers cần kết nối + instruction bằng natural language + LLM provider (Anthropic, OpenAI, Gemini, etc.)
- Output: Agent thực thi agentic loop — tự gọi tools từ MCP servers, trả kết quả tổng hợp
- Workflow: (1) Khai báo `Agent` với `server_names` và `instruction`; (2) `attach_llm` chọn model; (3) `generate_str` chạy agentic loop tự động; (4) Có thể compose nhiều agents theo patterns: fan-out (parallel specialists) → fan-in, orchestrator-worker, evaluator-optimizer, router
- Hỗ trợ swap `execution_engine` từ mặc định sang Temporal để có durable state (pause/resume/retry) mà không cần thay code

**Real example from README**:

```python
import asyncio
from mcp_agent.app import MCPApp
from mcp_agent.agents.agent import Agent
from mcp_agent.workflows.llm.augmented_llm_openai import OpenAIAugmentedLLM

app = MCPApp(name="hello_world")

async def main():
    async with app.run():
        agent = Agent(
            name="finder",
            instruction="Use filesystem and fetch to answer questions.",
            server_names=["filesystem", "fetch"],
        )
        async with agent:
            llm = await agent.attach_llm(OpenAIAugmentedLLM)
            answer = await llm.generate_str("Summarize README.md")
            print(answer)

asyncio.run(main())
```

**Install instructions**:

```bash
# Với uv (nhanh hơn)
uv add "mcp-agent"

# Hoặc pip
pip install mcp-agent

# Với LLM provider support cụ thể:
uv add "mcp-agent[anthropic]"
uv add "mcp-agent[openai, anthropic, google]"
```

**Problem it solves**: Xây agent kết nối với MCP servers thủ công đòi hỏi viết connection management, retry logic, và agentic loop từ đầu. `mcp-agent` cung cấp sẵn các composable patterns (fan-out, evaluator, router) và hỗ trợ durable execution qua Temporal mà không cần thay đổi workflow code.

**Existing alternatives**:

- Anthropic SDK trực tiếp với tool_use (không có sẵn patterns, phải tự viết orchestration)

**Repo signals**:
- Stars: 7,454
- Last commit: 2026-01-25
- Open issues: No info in README
- License: Apache 2.0
- Maintainer: lastmile-ai

**Solo-dev fit**: ⭐ × 4
Lý do: `pip install` một lệnh, Python thuần, docs đầy đủ tại docs.mcp-agent.com; last commit tháng 1/2026 nên không quá stale nhưng cần verify nếu dùng production.

### 1. `/goal` command + `claude agents` — Autonomous Multi-Turn Workflow Pattern

**URL nguồn**: https://x.com/ClaudeCodeLog/status/2053913638197416198 (X/Twitter) + https://code.claude.com/docs/en/changelog

**Type**: Technique

**What it actually does**:
- Input: Điều kiện hoàn thành viết bằng natural language (ví dụ: "All tests pass and the PR description is written")
- Output: Claude tự chạy nhiều turns liên tiếp, mỗi turn thực thi tool calls, cho đến khi `goal evaluator` xác nhận điều kiện đã đạt; kèm live overlay hiển thị elapsed time / turns / tokens
- Workflow: (1) Dùng `/goal <completion-condition>` để đặt mục tiêu; (2) Claude tự loop qua các bước cần thiết; (3) `claude agents` liệt kê tất cả sessions đang chạy/chờ input/đã xong; (4) Pause/resume background sessions qua `/resume`

**Real example from source**:

```
# Từ Claude Code Changelog 2.1.139 (May 11, 2026):
Added /goal command: set completion condition and Claude keeps working until met with live overlay.
Available in interactive mode, -p, and Remote Control.
Tracks: elapsed time, turns used, tokens consumed.

Hook enhancement: args: string[] field now spawns command directly without shell
Hook enhancement: continueOnBlock option for PostToolUse feeds rejection back to Claude
```

**How to apply**:

```bash
# Trong Claude Code session:
/goal All unit tests pass and PR description is drafted with breaking changes listed

# Xem tất cả sessions đang chạy:
claude agents

# Scope về thư mục cụ thể:
claude agents --cwd /path/to/project

# Resume session background:
/resume
```

**Problem it solves**: Trước 2.1.139, multi-step tasks (ví dụ: viết feature → run tests → fix failures → repeat) đòi hỏi developer phải ngồi đợi mỗi turn để approve bước tiếp theo. `/goal` cho phép đặt điều kiện xong việc và để agent tự chạy — không cần giám sát từng bước.

**Source signals**:
- Ngày đăng: 2026-05-11 (changelog 2.1.139), confirmed qua X/Twitter @ClaudeCodeLog
- Tác giả: Anthropic (official release)
- Platform: Official Claude Code Changelog

**Caution**: Theo changelog 2.1.143, bug đã fix: `/goal` evaluator trước đó có thể fire sai khi background shells hoặc subagents vẫn đang chạy — đã được fix trong 2.1.143 (May 15). Nên dùng >= 2.1.143.

**Existing alternatives**:
- Chạy `claude -p "..." --dangerously-skip-permissions` trong shell loop — không có built-in evaluator, phải tự viết check logic
- Dùng hooks `PostToolUse` + `continueOnBlock` (phức tạp hơn, phải viết script riêng)

**Solo-dev fit**: ⭐ × 5
Lý do: Có thể đặt `/goal all tests pass` trước khi đi ngủ, sáng dậy xem kết quả — zero additional setup, built-in vào Claude Code.

**Concrete action to try it**:
```bash
# Cập nhật Claude Code nếu < 2.1.143:
npm update -g @anthropic-ai/claude-code
# Trong một session đang làm việc:
/goal All TypeScript errors are fixed and `npm test` exits 0
```

**Recommendation**: Try now (zero setup — chỉ cần Claude Code >= 2.1.143)
