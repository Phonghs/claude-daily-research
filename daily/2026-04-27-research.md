### 2.1 Prompt Caching Deep Dive — Pitfalls & Best Practices
- **Sources**:
  - [Prompt Caching Guide (Claude API Docs)](https://platform.claude.com/docs/en/build-with-claude/prompt-caching)
  - [How Prompt Caching Actually Works in Claude Code](https://www.claudecodecamp.com/p/how-prompt-caching-actually-works-in-claude-code)
  - [Lessons from Building Claude Code: Prompt Caching Is Everything](https://x.com/trq212/status/2024574133011673516)

- **Mô tả**: Prompt caching cho phép reuse computed tokens từ previous requests, giảm chi phí input tokens 90% và latency 85% trên long prompts. Nhưng có nhiều pitfalls khiến cache bị invalidate và cost 5x.

- **Highlights**:
  - 💰 Cached input = 10% normal cost ($0.50/M vs $5/M)
  - ⚡ Long session (~100 turns): $50-100 → $10-19
  - 📏 Automatic vs explicit cache breakpoints (flexible control)
  - ⚙️ Supports tool use, system prompts, multi-turn conversations

- **Pitfalls**:
  - ❌ **Tool set change mid-conversation** — Most common cache killer. Adding/removing MCP tool = full cache invalidate
  - ❌ **Timestamp in system prompt** — Breaks cache on every turn
  - ❌ **Switching models** — Claude 3 Sonnet → Opus invalidates
  - ⚠️ **March 2026 caching bug** — 10-20x token inflation with no warning (đã fix)

- **Use case chính**: Long-running sessions, repeated file reads, bug-fix iterations, code review loops
- **Phù hợp**: ⭐⭐⭐⭐⭐ (5/5) — Hiệu quả massive, dễ implement, zero-config available
- **Recommendation**: **Nên thử ngay** — Enable via `cache_control` parameter, monitor cache hits, avoid tool changes mid-session

### 2.2 Claude Code Token Optimization — 18 Hacks Guide (2026)
- **Sources**:
  - [Claude Code Token Optimization: Full System Guide (2026)](https://buildtolaunch.substack.com/p/claude-code-token-optimization)
  - [7 Ways to Cut Your Claude Code Token Usage](https://dev.to/boucle2026/7-ways-to-cut-your-claude-code-token-usage-elb)
  - [18 Claude Code Token Management Hacks](https://www.mindstudio.ai/blog/claude-code-token-management-hacks)
  - [Reduce Claude Code token usage by 90%](https://medium.com/data-science-in-your-pocket/reduce-claude-code-token-usage-by-90-baa2a27b9ca3)

- **Mô tả**: Comprehensive guide covering context trimming, model selection strategy, compaction, .claudeignore setup để giảm token usage 40-70%.

- **Highlights**:
  - 📝 **CLAUDE.md optimization** — Keep under 500 tokens (5K CLAUDE.md = 5K tokens/turn)
  - 🔄 **Use /compact command** — Summarize history, free up context window
  - 🎯 **Model selection**: Sonnet (80% tasks), Haiku (cheap exploration), Opus (complex only)
  - 🚫 **Setup .claudeignore** — Prevent indexing build artifacts, lock files, generated code
  - 📊 **One task per chat** — Resets session, avoids context bloat
  - ⏱️ **Extended thinking costs** — Tens of thousands output tokens per request if enabled

- **Results**: Most developers save 40-70%, strategic switching → 70% cost reduction

- **Use case chính**: Solo dev managing API costs, long projects, repeated sessions
- **Phù hợp**: ⭐⭐⭐⭐⭐ (5/5) — Immediately applicable, no code changes, saves money
- **Recommendation**: **Nên thử ngay** — Apply .claudeignore + trim CLAUDE.md first (quick wins), then A/B test Haiku for exploration

### 2.3 Claude Code Hooks & Subagents Mastery
- **Sources**:
  - [Create custom subagents (Claude Code Docs)](https://code.claude.com/docs/en/sub-agents)
  - [Claude Code Hooks Mastery](https://github.com/disler/claude-code-hooks-mastery)

- **Mô tả**: Deep dive vào advanced Claude Code features: hooks (lifecycle events), subagents (isolated context), và 5 workflow patterns (sequential, operator, split-and-merge, agent teams, headless).

- **Highlights**:
  - 🎯 **Subagents** = specialized AI in own context (perfect for side tasks, dedup search output)
  - 🔗 **Hooks** = 25 lifecycle events (UserPromptSubmit, PreToolUse = security checkpoints)
  - 📐 **Operator Pattern** — One "brain" agent coordinates subagents (best for complex tasks)
  - 🤝 **Agent Teams** — Persistent group of specialized agents (beyond single task)
  - ⚙️ **YAML frontmatter** — Define subagents with description, prompt, tools, model, hooks

- **Use case chính**: Complex multi-step automation, task decomposition, security/validation gates
- **Phù hợp**: ⭐⭐⭐⭐ (4/5) — Powerful, requires learning curve, payoff = better task organization
- **Recommendation**: **Theo dõi** — Start with single subagent for isolated side tasks (search, analysis), learn operator pattern next
