- Workflow: Claude agent says "Check pending orders" → MCP translates to Squarespace API → returns structured order list with notes → agent can fulfill orders via MCP tool calls
- **Real example**: Query: "Show me low-stock products" → tool `squarespace_check_low_stock(threshold=10)` → returns inventory with reorder alerts

**Real example from README**:

```javascript
// List pending orders
squarespace_list_orders({ fulfillmentStatus: "PENDING" })
// Returns: [{orderId, customerName, items, status, createdDate}]

// Check inventory

// Returns: [{productId, name, currentStock, lowStockAlert}]

// Get revenue metrics
```

### 3. karanb192/claude-code-hooks

**URL**: https://github.com/karanb192/claude-code-hooks

**Type**: Hook collection with security focus

**What it actually does** (từ README):
- Input: 3 hook categories (Pre-Tool-Use, Post-Tool-Use, Notification) + configurable safety levels
- Output: Production-ready hook scripts that block dangerous commands, protect secrets, auto-stage files, send notifications → layered security without manual Claude prompting
- Workflow: Copy hook → register in `.claude/settings.json` → Claude Code auto-executes hook on event → prevents execution if safety rule violated → shows error to Claude
- **Real example**: User attempts `rm -rf ~` → Pre-Tool-Use hook fires → detects pattern → exits with code 2 → Claude sees error and asks for confirmation instead of executing immediately

**Real example from README**:

```bash
# Hook: block-dangerous-commands (Pre-Tool-Use)
# Detects and prevents: rm -rf /, fork bombs, > /dev/null shenanigans
# Configurable safety levels: critical, high, strict

# Copy to ~/.claude/hooks/
cp hook-scripts/pre-tool-use/block-dangerous-commands.js ~/.claude/hooks/

# Register in ~/.claude/settings.json:
# "hooks": {
#   "PreToolUse": [{
#     "command": "~/.claude/hooks/block-dangerous-commands.js",
#     "matcher": ".*"
#   }]
# }

# Result: Claude Code prevents dangerous commands, shows error, prompts user instead of executing
```

**Install instructions**:

```bash
git clone https://github.com/karanb192/claude-code-hooks

```

**Problem it solves**: Agents run unsafe commands without safeguards (fork bombs, mass deletes, credential theft) → developers either skip powerful features or expose risk. Hooks = deterministic safety layer → Pre-Tool-Use hooks block dangerous patterns before execution, Post-Tool-Use validates output, Notification hooks alert on permission requests → no more "did Claude just delete my repo?"
