### 2. thedotmack/claude-mem

**URL**: https://github.com/thedotmack/claude-mem

**Type**: SDK / Plugin

**What it actually does**:
- Input: Tool observations và results xảy ra trong một session Claude Code (file reads, bash outputs, API calls)
- Output: Semantic summaries được persist vào `~/.claude-mem/` và searchable ở các session sau
- Workflow: Session chạy → claude-mem captures tool outputs theo realtime → tạo semantic summary → session mới query memory → agent nhận lại context mà không cần user recap
- Hỗ trợ cross-tool: Claude Code, Gemini CLI, OpenCode qua cùng một install

**Real example from README**:
```bash
# Quick install cho Claude Code:
npx claude-mem install

# Cho Gemini CLI:
npx claude-mem install --ide gemini-cli

# Qua plugin marketplace:
/plugin marketplace add thedotmack/claude-mem
/plugin install claude-mem
```

Config tại `~/.claude-mem/settings.json`:
```json
{
  "CLAUDE_MEM_MODE": "code--zh"
}
```
(Hỗ trợ `code`, `code--zh`, `code--ja` và bất kỳ ISO-639-1 language code nào)

**Install instructions**:
```bash
npx claude-mem install
```

**Problem it solves**: Claude Code mất toàn bộ context sau mỗi session — developer phải paste lại project state từ đầu mỗi sáng. claude-mem tự động lưu và retrieve semantic memories mà không cần thao tác thủ công nào.
