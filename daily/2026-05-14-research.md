### 3. Kỹ thuật: Hook `args: string[]` Exec Form — Tránh quoting bugs trong shell hooks

**Source**: Claude Code Docs (https://code.claude.com/docs/en/hooks) — tính năng mới trong v2.1.139 (May 11, 2026), chưa được đề cập trong báo cáo May 12.

**Type**: Hook configuration technique

**What it actually does**:
- Input: Hook config JSON với field `args: string[]` thay vì dùng shell command string.
- Output: Claude Code spawn process trực tiếp (exec form) mà không qua shell — mỗi element trong `args` là một argument riêng biệt, không cần quote.
- Workflow: Khi `args` có mặt trong hook config → Claude Code resolve `command` như executable trên PATH → spawn thẳng với args vector → không có shell interpretation → `${CLAUDE_PLUGIN_ROOT}` và các placeholder được substitute vào string nhưng không bị shell parse lại.

**Real example from docs** (snippet từ official hooks reference):

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "node",
            "args": ["${CLAUDE_PLUGIN_ROOT}/scripts/format.js", "--fix"]
          }
        ]
      }
    ]
  }
}
```

Nếu không có `args`, Claude Code chạy `command` qua shell, và path có spaces hoặc special chars cần escape thủ công. Exec form loại bỏ vấn đề đó.

**How to apply**:
Mở `.claude/settings.json`, thêm `"args": [...]` vào bất kỳ hook nào đang dùng `command` dạng string. Đặc biệt hữu ích khi path chứa spaces hoặc khi truyền arguments có ký tự đặc biệt.

**Problem it solves**: Hooks chạy scripts với đường dẫn chứa spaces hoặc arguments đặc biệt hay bị broken do shell parsing. Exec form loại bỏ hoàn toàn shell layer — không cần escape, không có injection risk từ path placeholders.

**Existing alternatives**: Shell form (dùng quotes thủ công, dễ lỗi), wrapper scripts (thêm file, phức tạp hơn).

**Source signals**:
- Platform: Official Claude Code Docs (code.claude.com/docs/en/hooks)
- Feature released: v2.1.139, May 11, 2026
- Author: Anthropic engineering team

**Solo-dev fit**: ⭐ × 4
Reason: Drop-in thay thế trong config JSON; giải quyết bug hook thường gặp khi path có spaces mà không cần tạo wrapper script.

**Concrete action to try it**:
Tìm hook nào đang dùng `"command": "path/to/script.sh"`, đổi thành `"command": "/path/to/script.sh", "args": []` — nếu cần pass arguments, cho từng arg vào mảng riêng biệt.

**Recommendation**: Try now

### 4. Kỹ thuật: Kiểm tra Claude Code cache TTL đang là 5 phút hay 1 giờ qua JSONL logs

**Source**: GitHub issue #46829 (https://github.com/anthropics/claude-code/issues/46829) + XDA Developers (https://www.xda-developers.com/anthropic-quietly-nerfed-claude-code-hour-cache-token-budget/)

**Type**: Cost optimization / diagnostic technique

**What it actually does**:
- Input: File JSONL trong `~/.claude/projects/` — Claude Code ghi đầy đủ API interactions vào đây.
- Output: Xác nhận xem workspace của bạn đang dùng cache TTL 5 phút (ephemeral, default từ tháng 3/2026) hay 1 giờ (persistent).
- Workflow: Đọc JSONL → tìm object `usage.cache_creation` trong mỗi assistant response → nếu thấy cache write liên tục sau mỗi lần dừng >5 phút → đang dùng 5m TTL → cần điều chỉnh workflow.

Tác động: Cache TTL 5 phút nghĩa là mỗi lần pause >5 phút trong session (ví dụ đi uống cà phê, chờ tests chạy), toàn bộ cached context expire và phải re-upload ở mức write rate (1.25x thay vì read rate 0.1x). Trong thực tế, nhiều developer báo cáo chi phí tăng 15–53% sau thay đổi ngầm này.

**Real example** (từ GitHub issue và community findings):

```bash
# Đọc JSONL log để kiểm tra cache_creation objects:
cat ~/.claude/projects/$(ls -t ~/.claude/projects/ | head -1)/*.jsonl \
  | python3 -c "
import sys, json
for line in sys.stdin:
    try:
        obj = json.loads(line)
        usage = obj.get('usage', {})
        if 'cache_creation_input_tokens' in usage:
            print(f'cache_write: {usage[\"cache_creation_input_tokens\"]} tokens')
        if 'cache_read_input_tokens' in usage:
            print(f'cache_read:  {usage[\"cache_read_input_tokens\"]} tokens')
    except: pass
"
# Nếu thấy cache_write nhiều lần trong cùng session → TTL đang ở 5 phút
```

**How to apply**:
1. Chạy script trên để xác nhận TTL hiện tại.
2. Nếu bị 5m TTL: giảm pause trong session (duy trì activity <5 phút), hoặc tách workflow thành batch runs ngắn thay vì sessions dài có idle time.
3. Cân nhắc dùng `cache_control: {"type": "ephemeral"}` một cách có chủ đích trong API calls thay vì mặc định.

**Problem it solves**: Anthropic đổi cache TTL ngầm từ 60 phút xuống 5 phút tháng 3/2026 mà không có announcement rõ ràng. Nhiều developer tiếp tục dùng workflow cũ (session dài, pause giữa chừng) và bị overcharged mà không biết nguyên nhân.

**Existing alternatives**: Xem billing dashboard (chỉ cho tổng số, không chi tiết tại sao), dùng Anthropic API console stats (cần truy cập dashboard, không offline).

**Source signals**:
- Platform: GitHub (anthropics/claude-code issue #46829) + XDA Developers
- Discovery date: tháng 3/2026 (cache change) + April-May 2026 (community analysis)
- Evidence: JSONL logs trên local machine — xác minh được offline, không cần API call

**Solo-dev fit**: ⭐ × 5
Reason: Script đọc file local, chạy offline, xác nhận ngay xem mình có bị ảnh hưởng không — không cần login hay dashboard.
