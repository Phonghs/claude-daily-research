### 4. `/goal` Command + Agent View: Autonomous Multi-Turn Agent Sessions

**Type**: Built-in Claude Code feature (v2.1.139)

**What it actually does**:
- Input: `/goal "<điều kiện hoàn thành>"` — mô tả trạng thái kết thúc mong muốn bằng ngôn ngữ tự nhiên
- Output: Claude tự động chạy multi-turn cho đến khi điều kiện được đáp ứng, với live overlay hiển thị elapsed time, turn count, và token usage
- Workflow: Gõ `/goal "All tests pass"` → Claude bắt đầu loop tự động → mỗi turn: đọc output, sửa lỗi, chạy lại → dừng khi goal được đáp ứng
- Agent View (`claude agents` command, Research Preview): unified dashboard terminal hiển thị tất cả background sessions đang chạy song song, kèm model và effort level của từng session

**Real example từ changelog v2.1.139**:

```
- New agent view (Research Preview) with `claude agents` command
- `/goal` command for setting completion conditions
- Hook `args: string[]` field for direct command spawning
- Various plugin and session management improvements
```

**How to apply**:

```bash
# Trong Claude Code session:
/goal "All unit tests pass and there are zero TypeScript compilation errors"

# Trong terminal khác, xem progress:
claude agents

# Xem sessions trong thư mục cụ thể:
claude agents --cwd ./src
```

**Problem it solves**: Trước đây phải ngồi theo dõi và approve từng turn cho long-running tasks như "fix all failing tests" hoặc "resolve all lint errors." `/goal` cho phép set điều kiện một lần và Claude tự chạy đến khi xong — không cần `--dangerously-skip-permissions` vì vẫn có completion condition rõ ràng.

**Existing alternatives**: `--dangerously-skip-permissions` flag (bỏ qua tất cả permissions nhưng không có completion condition — không an toàn), hook-based automation (cần viết hook scripts phức tạp hơn).

**Source signals**:
- Publish date: May 12, 2026
- Author: Anthropic (Claude Code team)
- Platform: Claude Code Releases — v2.1.139 (https://github.com/anthropics/claude-code/releases)

**Solo-dev fit**: ⭐ × 5
Reason: Zero setup required — built vào Claude Code, chỉ cần gõ `/goal` và mô tả điều kiện bằng ngôn ngữ tự nhiên, không cần config hay plugin.

**Concrete action to try it**:
Trong project có failing tests:
```bash
/goal "npm test exits with code 0 — all tests passing"
```
Rồi mở terminal thứ hai: `claude agents` để xem live progress.

### 5. `worktree.bgIsolation: "none"` — Background Sessions Sửa Working Copy Trực Tiếp

**Type**: Built-in Claude Code setting (v2.1.143)

**What it actually does**:
- Input: Thêm `"worktree": { "bgIsolation": "none" }` vào `~/.claude/settings.json`
- Output: Background sessions edit trực tiếp trên working copy thay vì clone vào git worktree riêng biệt
- Workflow: Không có setting này → background session tạo worktree riêng → changes không hiển thị trong IDE cho đến khi merge. Với setting này → background agent edit trực tiếp, IDE hot-reload ngay lập tức, changes visible real-time
- Cũng trong v2.1.143: projected context cost estimates (per-turn và per-invocation token) trong `/plugin` marketplace browse pane; plugin dependency enforcement khi `claude plugin disable`

**Real example từ changelog v2.1.143**:

```
- Added worktree.bgIsolation: "none" setting for background sessions
  to edit working copy directly without EnterWorktree
- Added projected context cost (per-turn and per-invocation token estimates)
  to the /plugin marketplace browse pane
- Plugin dependency enforcement: claude plugin disable now refuses
  when another enabled plugin depends on the target
- Fixed stop hooks blocking repeatedly looping forever
  (now ends with warning after 8 consecutive blocks)
- PowerShell tool now passes -ExecutionPolicy Bypass by default
```

**How to apply**:

```bash
# Thêm vào ~/.claude/settings.json:
{
  "worktree": {
    "bgIsolation": "none"
  }
}

# Opt-out PowerShell bypass nếu cần:
# CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1
```

**Problem it solves**: Background sessions mặc định dùng git worktrees riêng biệt: changes không hiển thị trong IDE real-time và cần sync/merge thêm bước. Setting này giúp background agent hoạt động như foreground session khi developer muốn xem changes ngay lập tức.

**Existing alternatives**: `EnterWorktree` hook (workaround phức tạp hơn, cần viết hook script thủ công).

**Source signals**:
- Publish date: May 15, 2026
- Author: Anthropic (Claude Code team)
- Platform: Claude Code Releases — v2.1.143 (https://github.com/anthropics/claude-code/releases)

**Solo-dev fit**: ⭐ × 4
Reason: Một dòng config trong settings.json, không cần restart, áp dụng cho tất cả background sessions ngay lập tức.

**Concrete action to try it**:
```bash
# Thêm vào ~/.claude/settings.json:
{ "worktree": { "bgIsolation": "none" } }
```
Chạy một background task (`claude --bg "fix all linting errors"`) và mở IDE để xem changes xuất hiện real-time.
